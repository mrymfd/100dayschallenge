_Contributing to 100 Days Challenge_
Thank you for your interest in contributing! This guide explains how to contribute to the project and make your contributions clear and helpful.


_Low to Contribute / Workflow_
🍴 Fork the Repository

Click the “Fork” button on GitHub to create your own copy.

💻 Clone Your Fork

git clone https://github.com/your-username/100dayschallenge.git

cd 100dayschallenge

🌿 Create a Branch

Use descriptive branch names:

Features: feature/your-feature-name

Bug fixes: bugfix/issue-number

Documentation: docs/update-readme

git checkout -b feature/add-new-challenge

🛠 Make Changes

Add code in the src/ folder

Add resources in resources/

Update documentation in docs/ or README.md

📝 Commit Changes
Use clear messages and gitmojis:

git commit -m "✨ Add new challenge progress tracker"

git commit -m "🐛 Fix typo in README"

git commit -m "📝 Update CONTRIBUTING guidelines"

📤 Push Your Branch

git push origin your-branch-name

🔗 Open a Pull Request (PR)

Provide a clear description of your changes

Link related issues (if any)

Add labels if appropriate (enhancement, bug, documentation)

_Coding Style / Standards_

Use clear and descriptive variable names

Follow consistent indentation (4 spaces)

Keep functions small and focused

Comment your code when necessary

Write tests for new features if possible

❓ FAQs / Common Issues

Q: Can I contribute without knowing all the code?

A: Yes! You can help with documentation, small bug fixes, or ideas.


Q: How do I suggest a new feature?

A: Open an issue with your idea and discuss it before coding.


Q: How should I write commit messages?

A: Keep them short and meaningful. Use gitmojis if you like.


🧪 Testing and Quality
Run any tests before committing (if applicable)

Follow consistent coding style and project conventions

Keep code readable and maintainable
